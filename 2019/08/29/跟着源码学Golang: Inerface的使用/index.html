golang-source-code-interface

发表于 2019-08-29 | 分类于 Golang

听说过鸭子类型(Duck typing)吗？没听过也不要紧，通俗来讲，就是任何一个物体，只要它走路像鸭子且能嘎嘎叫，就可以称之为鸭子。学过Python面试时肯定常被问及什么是迭代器，这里的迭代器就可以称为一种“鸭子”，因为只要实现了签名符合要求的__next__和__iter__方法则可称为迭代器。

< !–more– >

为什么我要提及鸭子类型，这明明是动态语言里的东西啊。因为Go也使用类似的思想作为它的类型系统，称之为结构类型系统(Structural type systems)，它是在编译时决定类型的兼容性和等效性，而鸭子类型则是在运行判断的。相较于Java这种基于名字的类型系统，它不用显示地定义继承于哪些父类，更加灵活方便

利用这种思想，我们可以做许多有趣的事情，Go使用Interface实现相关用法

现在让我们来Go标准库里的sort包里是如何利用Interface来优雅地解决对不同类型数据的排序的

  1. 首先，要让一个容器对象可排序，它需要什么特征(方法)呢？

    • 通过分析，我们仅需下述3个方法则可以满足所有排序算法所需
    • Len() int, 返回容器中的元素个数
    • Less(i, j int) bool, 元素比较功能
    • Swap(i, j int), 元素位置交换功能

  2. 现在如果我们需要排序一个类型为int的slice，应该怎么办呢？源码中已给出代码

  3. 实际编程中，我们应该注意些什么呢？

    • 接口尽量小，Go推崇小接口，例如io.Writer接口就只需要一个Write(p []byte) (n int, err error)方法
    • 对于大接口，我们可以用多个小接口拼接而成
